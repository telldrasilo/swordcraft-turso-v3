# Техническое задание: интерфейс планирования крафта по рецепту

Документ описывает **фактическую реализацию** экрана планирования крафта в приложении на момент составления. Предназначен для отдельной проработки дизайна и UX без привязки к внутренней документации проекта. Перечислены пользовательские сценарии, порядок блоков на экране, источники данных и правила валидации в терминах поведения интерфейса.

---

## 1. Назначение экрана

Игрок формирует **план крафта** перед запуском процесса: выбирает форму оружия (рецепт), назначает **каталожные материалы** на каждую часть клинка, при необходимости включает **цепочку подготовки сырья** (переплавка, пиление и т.п.), выбирает до **трёх боевых техник** крафта, видит **сводку потребности в ресурсах** и **прогноз характеристик и качества** будущего оружия. Кнопка «Начать крафт» активна только если выполнены все проверки (наличие сырья или согласие на закупку с достаточным золотом, отсутствие блокировок по материалам и совместимости техник).

Корневой компонент интерфейса: `CraftPlanner` в модуле `src/components/forge/craft-v2/craft-planner.tsx`. Дочерние блоки лежат в каталоге `src/components/forge/craft-v2/planner/`.

---

## 2. Входные данные экрана (пропсы)

Снаружи в планировщик передаётся состояние игрока и колбэки. Для дизайна важно понимать ограничения:

- **Уровень игрока** — фильтр доступных рецептов и участвует в расчёте прогноза как уровень кузнеца (если отдельный уровень мастерства не передан).
- **Инвентарь** — классический набор ресурсов (руда, уголь, готовые слитки и прочее в модели `Resources`).
- **Склад материалов** — отдельное отображение количеств по **каталожным id** материалов; учитывается вместе с инвентарём при сортировке кандидатов и при проверке «хватает ли на крафт».
- **Золото** — для отображения возможности докупки и кнопки старта при включённой закупке.
- **Списки разблокировок** — id рецептов, id **боевых** техник (чекбоксы «Техники»), id **техник обработки материала** (плавка и др.; дополняются правилами по уровню внутри вспомогательной функции разблокировок планировщика).
- **Знания по материалам** — экспертиза и флаги доступа к материалу в кузнице; без этого материал не попадает в выбор.
- **Цены материалов** — для отображения в селекторе материалов (покупка).
- **Активный заказ NPC** (опционально) — награда в золоте и уже взятый аванс; нужен для опции «взять аванс на закупку» при нехватке золота.

Режим **фиксированного рецепта**: если передан идентификатор одного рецепта, сетка выбора формы не показывается; вместо неё — одна карточка с названием и описанием этого рецепта (типично вкладка с одной формой, например «алтарь»).

## 3. Порядок блоков на экране (сверху вниз)

1. Выбор рецепта **или** карточка фиксированного рецепта.
2. Блок **«Материалы»** — по одной строке на каждую часть рецепта: слева выбор материала части, справа (на широком экране) панель обработки сырья, если для выбранного каталожного материала есть техники обработки; на узком экране вместо правой колонки — кнопка, открывающая нижнюю шторку с той же панелью.
3. Блок **«Материалы»** (вторая карточка с иконкой пакета) — проверка запасов, список требований, опционально закупка и аванс. Появляется, когда выбран рецепт и хотя бы по одной части выбран материал.
4. Аккордеон **«Техники»** — выбор до трёх боевых техник. Показывается при выбранном рецепте (даже если материалы ещё не выбраны полностью).
5. **«Прогноз результата»** — диапазоны статов, качество, подсказки. Появляется только когда есть прогноз (есть рецепт и хотя бы один выбранный материал в плане).
6. Кнопка **«Начать крафт»** — на всю ширину; текст и доступность зависят от проверок.

Между блоками используются вертикальные отступы и анимация появления при смене рецепта или состава плана.

---

## 4. Выбор рецепта

### 4.1. Обычный режим

- Заголовок карточки: «Выберите рецепт»; подзаголовок с **числом доступных** рецептов.
- Список рецептов строится так: из полного каталога форм отбираются рецепты, у которых выполняется хотя бы одно из условий: рецепт явно в списке разблокированных у игрока; или уровень игрока не ниже требуемого уровня рецепта; или выполняется разбор условия разблокировки из текста (паттерн уровня кузнеца); или у рецепта нет жёсткого условия и редкость источника «обычная». Реализация: функция `getAvailableRecipes` в модуле `src/data/recipes/index.ts`.
- Отображение: сетка карточек (одна колонка на мобильном, до трёх на широком экране) внутри прокручиваемой области с ограничением высоты (чтобы список не раздувал страницу целиком).
- **Карточка рецепта** (`RecipeCard`): иконка по типу оружия (меч, кинжал, топор, булава, копьё, молот и запасной вариант), название рецепта, строка метаданных: число **частей** и **требуемый уровень** (если не задан — отображается как 1). Выбранная карточка подсвечивается (рамка, фон, галочка). Клик по доступной карточке выбирает рецепт.

### 4.2. Поведение при смене рецепта

При выборе другого рецепта **сбрасываются**: выбранные материалы по частям, настройки цепочек обработки сырья по частям, закрывается шторка обработки если была открыта. **Не сбрасывается** явно в коде набор выбранных боевых техник — он остаётся прежним до ручного изменения игроком (дизайнеру стоит учесть возможную несовместимость с новым рецептом — валидация отдельно, см. раздел 9).

### 4.3. Режим фиксированного рецепта

Сетка выбора скрыта. Показывается одна карточка с **названием и описанием** уже известного рецепта. Идентификатор рецепта задаётся при монтировании планировщика.

## 5. Выбор материалов по частям

### 5.1. Карточка «Материалы» (первая)

- Заголовок с иконкой; описание: «Выберите материал для каждой части».
- Переключатель **«Только в наличии»**: если включён, в списке кандидатов для каждой части остаются только материалы, у которых суммарное количество в инвентаре и на складе **не меньше** минимального количества части из рецепта (`minQuantity`).

### 5.2. Селектор части (`PartMaterialSelector`)

Каждая часть рецепта — отдельный сворачиваемый блок.

**Заголовок свёрнутого состояния:** имя части (из рецепта); под ним либо «Выберите материал», либо **имя выбранного каталожного материала**; слева эмодзи-категория (металл, минерал, дерево, кожа) или «вопрос», если не выбрано.

**Развёрнутое состояние:** сетка карточек кандидатов (две колонки). Кандидаты формируются так:

- Берётся список материалов, допустимых для **данной части** и **категорий**, заданных рецептом для этой части (логика кандидатов: `getForgePartMaterialCandidates`).
- Отфильтровываются материалы, которые **нельзя списать** в крафте как каталожную единицу (внутренняя политика расхода в кузнице).
- Отфильтровываются материалы, **закрытые** для кузницы по знаниям игрока.
- Сортировка «умная»: учитываются инвентарь, склад, знания, рецепт, часть, уровень кузнеца и **доминирующее свойство** части рецепта (влияет на приоритет в списке).

На карточке кандидата отображаются: редкость (цветовая метка), имя, краткая числовая информация по ключевым свойствам части (доминирующее и вторичное свойство из рецепта), индикатор количества на складе/в инвентаре, цена покупки если есть. Есть тултип сравнения с текущим выбором (дельта к прогнозу свойств материала).

**Выбор:** клик по карточке задаёт каталожный id материала для части.

### 5.3. Связанная логика обработки сырья при выборе материала

Если для выбранного каталожного материала существует хотя бы одна **доступная игроку** техника обработки (плавка дерева, руды и т.д.):

- В состояние плана для этой части **автоматически** записывается режим цепочки **включён** и выбирается **первая** техника из отфильтрованного списка доступных (игрок может затем выключить цепочку или сменить технику в панели обработки).

Если доступных техник обработки нет — запись по этой части в плане цепочки **удаляется**.

## 6. Панель обработки сырья по части

Отображается только если для выбранного материала части есть хотя бы одна доступная техника обработки.

**Десктоп (широкая вёрстка):** панель справа от селектора материала в той же строке сетки.

**Мобильный / узкий:** панель не в строке; показывается кнопка «Обработка сырья», открывающая **нижнюю шторку** на весь экран с тем же содержимым. В шапке шторки: общий заголовок «Техника обработки», описание с именем части; если цепочка включена и выбрана конкретная техника — дополнительная строка с **именем техники** из игрового справочника.

**Содержимое панели (смысловые состояния):**

- Пока цепочка **выключена** (в плане нет записи для части): нейтральный текст о том, что доступно N вариантов подготовки и нужно включить переключатель; **не показывается** имя или описание конкретной техники как будто она уже выбрана.
- Цепочка **включена**: крупно **имя техники** (каноническое из данных); ниже компактный бейдж типа постройки (плавка, дерево, камень, кожа и т.д.) и короткая подсказка; затем описание техники; переключатель вкл/выкл с подписью, зависящей от типа постройки (например «Брать металл из руды», «Готовить доски у горна»); тултип с перечислением этапов цепочки и рецепта переработки; при **нескольких** техниках — выпадающий список выбора варианта; при **одном** варианте имя всё равно видно в шапке панели без селекта; внизу текст про списание со склада по рецепту переработки вместо готовой заготовки.

Отключение переключателя **удаляет** из плана запись обработки для этой части (каталожный материал части остаётся выбранным — игрок планирует тратить уже готовую заготовку, без цепочки у горна).

## 7. Расчёт и отображение потребности материалов на крафт

Отдельная карточка **«Материалы»** (вторая по счёту, с иконкой пакета) строит результат функции проверки склада `checkInventoryForCraft` в модуле `src/lib/craft/inventory-check.ts`.

**Вход в проверку:** рецепт; для каждой части, где выбран материал — пара «каталожный id материала» и количество `minQuantity` из рецепта; текущий инвентарь и склад; опционально карта «план обработки» по частям (режим переработки из сырья меняет то, **какие** ресурсы требуются — руда и уголь вместо слитка и т.п.).

**Выход, отражаемый в UI:**

- Если нет ни одной строки требований и нет блокирующего сообщения — **карточка не рендерится** (пустой результат).
- Иначе заголовок с индикатором: достаточно ли всего для крафта (зелёная / красная иконка, бейдж «Не хватает»).
- Текст **блокировки горна** (если возвращён `forgeSpendBlockReason`) — отдельным красным абзацем (например запрет списания определённого материала в контексте крафта).
- Секция **«Требуется сырья»**: список строк «название ресурса — доступно/нужно», цвет если не хватает.
- Строка **угля** как топлива, если переработка его требует — в том же формате достаточности.
- Раскрывающийся блок **«Детализация по частям»**: для каждой части имя части, выбранный материал, вложенный список «какое сырьё в каком количестве» для этой части.

**Закупка:** если не хватает ресурсов, но система считает, что недостающее **можно купить** (`canPurchaseMissing`), показывается чекбокс «Закупить недостающие материалы». При включении — список позиций к покупке с ценами и суммарная стоимость; если золота не хватает — предупреждение с числами «нужно / есть».

**Аванс по заказу:** если активен заказ NPC, не хватает золота на закупку, все позиции помечены как доступные к покупке и переданы данные заказа — показывается второй чекбокс про **аванс** на закупку (ограничение до половины награды заказа с учётом уже взятого аванса; текст подсказки рассчитывается в компоненте).

Карточка показывается при выбранном рецепте и **ненулевом** числе выбранных материалов по частям (хотя бы одна часть заполнена).

## 8. Выбор общих (боевых) техник крафта

Блок **«Техники»** — сворачиваемый (`TechniqueSelector`).

- В списке участвуют только техники, чьи id есть во **внешнем** списке разблокированных у игрока; источник данных — полный каталог техник, фильтрация по id.
- Заголовок аккордеона: «Техники»; подзаголовок в свёрнутом виде — либо «Не выбраны», либо **имена выбранных** через запятую; справа бейдж **число выбранных из трёх** и стрелка раскрытия.
- В развёрнутом виде — список чекбоксов: имя техники, описание. Можно выбрать **не более трёх**. Попытка выбрать четвёртую блокируется отключением чекбокса у невыбранных.
- Эти техники — **не** техники обработки руды/дерева; они влияют на расчёт характеристик и проходят отдельную валидацию совместимости с техниками обработки и планом материалов (при ошибке кнопка старта остаётся неактивной с текстом причины из валидации).

Блок виден при выбранном рецепте **независимо** от полноты выбора материалов.

## 9. Прогноз результата

### 9.1. Когда показывается

Прогноз считается только если выбран рецепт и **хотя бы один** материал выбран в `selectedMaterials` (фактически при ненулевом количестве ключей в объекте выбора). Компонент панели прогноза получает ненулевой объект прогноза из функции `calculateForecast` в `src/lib/craft/calculator.ts`.

### 9.2. Входы расчёта

- Рецепт.
- Назначение материалов по частям: только те части, где выбран материал; количество по части — из рецепта (`minQuantity`).
- Массив **объектов техник** — по id из выбранных боевых техник, несуществующие id отбрасываются.
- **Уровень кузнеца** для прогноза: отдельный проп «уровень мастера» если передан, иначе уровень игрока.
- Карта **экспертизы по id материала** (из знаний игрока).
- Два агрегата по техникам обработки материала в плане: **суммарный бонус к качеству** от выбранных цепочек и **сжатие разброса** прогноза (чем выше, тем уже диапазоны при прочих равных). Оба считаются из плана обработки и назначения материалов.

### 9.3. Содержимое панели «Прогноз результата»

- Заголовок карточки с иконкой.
- **Левая колонка (на широком экране):** для статов **атака**, **прочность**, **вес**, **потенциал души** — строки с иконкой, подписью и **диапазоном** значений (минимум–максимум), отражающим неопределённость крафта.
- **Правая колонка:** крупный индикатор **качества** (ранг/цвет/подпись из модели прогноза); блок **«Слабое место»** — какой из атаки/прочности/веса имеет **наибольший разброс** (наибольшая неопределённость).
- Если средняя экспертиза по **выбранным** материалам достигает порога «высокой» или «максимальной» вехи — зелёная подсказка о дополнительном бонусе к качеству в расчёте (пороги задаются константами игры).
- Если **точность прогноза** ниже 100%, показывается предупреждающий блок с процентом точности и советом повысить экспертизу материалов; цветовая интенсивность зависит от того, ниже ли порог 70%.

Средняя экспертиза для подсказок считается по **всем частям рецепта**, для которых выбран материал: среднее арифметическое значений экспертизы из знаний.

## 10. Кнопка «Начать крафт»

Кнопка на всю ширину, в конце цепочки блоков. Активна только если выполняются **все** условия:

- Выбран рецепт.
- Для каждой **обязательной** части рецепта выбран материал (части с флагом «опциональная» могут быть пустыми).
- Для каждого выбранного материала: разрешён расход в крафте горна; материал **разблокирован** для кузницы по знаниям.
- Проверка склада не возвращает блокирующую причину по горну.
- Валидация совместимости **техник обработки** и **боевых техник** с планом материалов успешна (иначе на кнопке текст ошибки из валидации).
- Либо `checkInventoryForCraft` сообщает, что **можно крафтить** с текущими запасами, либо включена закупка недостающего и **хватает золота** на сумму закупки (с учётом опции аванса по заказу: доступный остаток аванса добавляется к «имеющемуся» золоту для сравнения).

При нажатии родительскому обработчику передаётся объект: id рецепта; материалы по частям с количествами из рецепта; массив id выбранных **боевых** техник; флаги закупки и аванса; карта обработки по частям **если непустая** (режим и id техники обработки на часть).

Если кнопка неактивна, на ней отображается иконка предупреждения и краткий текст: либо причина валидации техник, либо общая фраза про недостаток материалов.

## 11. Состояние и согласованность данных (для проектирования)

- Идентификаторы **боевых техник** и **техник обработки материала** берутся из **разных** справочников; в UI не смешиваются в одном списке.
- План обработки хранится отдельно от выбора каталожного материала части; сброс материала пересчитывает план обработки для этой части.
- Прогноз и проверка склада используют **согласованные** представления назначения материалов и плана обработки; изменение переключателя цепочки или техники обработки меняет и прогноз, и строки требований в карточке проверки.
