# План исправления системы тегов событий

## Выявленные проблемы

### 1. Ресурсы в случайных событиях
- **Проблема:** В событиях генерируется слишком мало ресурсов (1-5 единиц)
- **Проблема:** В событиях даётся золото и слава — они должны выдаваться только по завершении миссии
- **Проблема:** Ресурсы не всегда берутся из пула доступных ресурсов локации

### 2. Прочность оружия
- **Проблема:** "Урон оружию" — неудачное название, должно быть "потеря прочности"
- **Проблема:** Нет баланса: оружие должно выдерживать 3-4 миссии до ремонта (до падения до порога в 20 единиц)
- **Проблема:** Потеря прочности не рассчитывается динамически в зависимости от maxDurability оружия

### 3. Коэффициенты балансировки
- **Проблема:** Падение прочности не зависит от коэффициентов балансировки
- **Проблема:** Душа войны не зависит от коэффициентов балансировки

## План исправления

### Этап 1: Исправление генерации ресурсов в событиях ✅

- [x] **1.1** Удалить генерацию золота и славы из `generateEventEffects()`
  - Золото и слава должны выдаваться только как награда за миссию
  - Убрать `effects.gold` и `effects.glory` из событий

- [x] **1.2** Использовать пул ресурсов локации для генерации материалов
  - Читать `location.resources` для получения списка доступных материалов
  - Использовать `minQuantity` и `maxQuantity` из конфигурации ресурса
  - Учитывать `rarity` и `baseWeight` при выборе материала

- [x] **1.3** Увеличить количество ресурсов в событиях
  - Позитивные события: 60% шанс на ресурсы, количество по minQuantity-maxQuantity локации
  - События выбора: 45% шанс на ресурсы, увеличенное количество
  - Учитывать `quantityMultiplier` из GLOBAL_BALANCE_CONFIG

### Этап 2: Система прочности оружия ✅

- [x] **2.1** Рассчитать базовый коэффициент потери прочности за миссию
  - Цель: оружие должно выдержать 3-4 миссии до падения до порога ремонта (20 единиц)
  - Формула расчёта коэффициента: `lossRate = (maxDurability - repairThreshold) / avgMissionsCount`
  - Где avgMissionsCount = 3.5 (среднее между 3 и 4 миссиями)

- [x] **2.2** Переработать формулу потери прочности
  - Базовая потеря за миссию: `lossRate` (рассчитывается динамически от maxDurability)
  - Формула: `durabilityLoss = lossRate * difficultyModifier * durationModifier`
  - Порог ремонта = 20 единиц (когда оружие требует починки)

- [x] **2.3** Заменить "damage" на "durabilityLoss" в эффектах
  - Переименовать поле в интерфейсе GeneratedEvent
  - Обновить UI для отображения потери прочности

- [x] **2.4** Добавить расчёт потери прочности в зависимости от параметров миссии
  ```
  lossRate = (weapon.maxDurability - repairThreshold) / 3.5
  difficultyModifier:
    - easy: 0.7
    - normal: 1.0
    - hard: 1.4
    - extreme: 2.0
  durationModifier: duration / 3600 (нормализация к 1 часу)
  finalLoss = lossRate * difficultyModifier * durationModifier
  ```
