# Карточка квеста «Эхо забытой кузни» (Особые задания)

Рабочий документ для визуального и UX-рефакторинга блока в `Экспедиции → Особые задания`. Источник правды по шагам и текстам: `src/data/quests/forgotten-forge.ts`, `src/types/forgotten-forge-quest.ts`, компонент `src/components/guild/forgotten-forge-quest-card.tsx`.

---

## 1. Стилистика проекта

- **Контейнер:** `Card` с классом `card-medieval`, акцентная обводка `border-amber-900/35` (активный квест), при необходимости тот же паттерн, что у `RecoveryQuestCard` (иконка в квадрате, `Progress` с `h-2 bg-stone-800`).
- **Иконки:** **Lucide** (`ScrollText`, `MapPin`, `Hammer`, `Landmark` / `Sparkles` для алтаря, `MessageSquare` / `ScrollText` для архивариуса), без эмодзи в продакшен-UI — единообразие с гильдией и кузницей.
- **Типографика:** заголовки `text-amber-200`, вторичный текст `text-stone-400` / `text-stone-500`, цель квеста — `text-stone-200` с `leading-relaxed`.
- **Прогресс:** компонент `Progress` из `@/components/ui/progress` (как на карточке восстановления оружия), процент подписью справа или под полосой мелким шрифтом.

## 2. Макет карточки (десктоп; мобилка — колонка, те же блоки)

```
┌──────────────────────────────────────────────────────────────┐
│ [ScrollText] Эхо забытой кузни          [Бейдж: статус]      │
│ Краткий анонс из FORGOTTEN_FORGE_CARD_BLURB (1–2 строки)     │
├──────────────────────────────────────────────────────────────┤
│ Шаг N · всего этапов ················ [████████░░]  NN%      │
├──────────────────────────────────────────────────────────────┤
│ Текущая цель                                                 │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ Текст из FORGOTTEN_FORGE_PROGRESS_LINES[step]            │ │
│ │ (учёт waitingForCraftAfterPhase2 на шаге 9 — см. ниже)   │ │
│ │ [ Один главный CTA: иконка + подпись ]                   │ │
│ └──────────────────────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────┤
│ [Опционально] Страховка / очистка чаши / Ансельм             │
│ (только на шагах 3, 5, 6 при awaiting_expedition)            │
├──────────────────────────────────────────────────────────────┤
│ [Вторично] Поговорить с архивариусом (outline)               │
├──────────────────────────────────────────────────────────────┤
│ [DEV] перезапуск / форвард цепочки                           │
└──────────────────────────────────────────────────────────────┘
```

**Принципы:**

- Один **основной** призыв к действию в блоке цели (куда идти *сейчас*). Вторичный — только диалог с архивариусом.
- Подсказка «Следующая цель экспедиции: …» дублирует данные из текста цели; при сжатом макете её можно убрать или показывать только если CTA ведёт в экспедиции.

## 3. Прогресс и счётчик шага

- Сценарий v2: шаги **`0` … `FORGOTTEN_FORGE_QUEST_STEP_MAX`**, сейчас `STEP_MAX = 18` (`src/types/forgotten-forge-quest.ts`). В строке прогресса использовать формулировку в духе: **«Этап N из 19»** (индексы 0–18) или **«Шаг N / 18»** с явной легендой в коде — зафиксировать один вариант, чтобы не путать игрока.
- **Полоса:**  
  `percent = Math.round((step / STEP_MAX) * 100)` для активного квеста; для `status === 'completed'` — `100%`.  
  Шаг `04` при `STEP_MAX = 18` ≈ 22%; шаг `9` ≈ 50% (в документе раньше фигурировало «16 шагов» — это устарело относительно v2).
- Текстовая строка из `getForgottenForgeProgressLine` остаётся источником сюжетной формулировки; визуально она живёт **внутри** блока «Текущая цель», а не как отдельная цитата с боковой границей (или граница только у блока целиком).

## 4. Главный CTA (динамика)

Зависит от шага, `forgottenForgePhase` и флага `waitingForCraftAfterPhase2`.

| Куда ведём | Подпись (пример) | Действие в store |
|------------|------------------|------------------|
| Экспедиции | «К экспедициям» | Уже есть: `onGoToExpeditionsTab` |
| Кузница, крафт | «К крафту» | `navigateToForgeTab('craft')` (+ `setCurrentScreen('forge')` уже внутри навигации) |
| Зачарования / стройка алтаря | «К алтарю» | `setCurrentScreen('altar')` |

**Ориентир по шагам (уточнить в коде хелпером, не дублировать магические числа в разметке):**

- Экспедиции: шаги с ожиданием похода по `getForgottenForgeExpeditionExpectation` (1–6, 11–13, 15) при `forgottenForgePhase === 'awaiting_expedition'`.
- Алтарь: шаги 8, 14, 16, 17 и любые промежутки, где цель — фаза строительства (см. строки прогресса в `forgotten-forge.ts`).
- Крафт: шаг 9, если активен сценарий «ждём крафт после фазы II» (`waitingForCraftAfterPhase2`), иначе для шага 9 — приоритет алтаря (фаза II), если это следует из логики квеста.
- Шаги «только диалог» (например 0, 7): главный CTA можно не показывать или заменить на «К архивариусу» (открытие дока сообщений), чтобы не плодить кнопки.

Итог: вынести **`getForgottenForgeQuestCardPrimaryCta(q, phase)`** в `src/lib/quests/` или рядом со слайсом, вернуть `{ label, icon, onNavigate }` или `{ kind: 'expedition' | 'forge' | 'altar' | 'none' }`.

## 5. Опциональные блоки (без смены логики)

- **Шаг 3, awaiting expedition:** страховка **«Оформить шахтную страховку (80 золота)»** — короткий заголовок, в подписи: «Списывается при старте экспедиции. Влияет только на текст отчёта архивариуса, не на добычу.» Переключатель `Switch` как сейчас.
- **Шаг 5:** «Очистка чаши» — подзаголовок нейтральнее: **«Выберите способ очистки фокусирующей чаши»**, кнопки «Магия» / «Физически».
- **Шаг 6:** **«Сделка с Ансельмом»** — кнопки «Сделка» / «Кража»; краткая подсказка, что выбор отражается в диалоге/отчёте.

Все три блока — отдельная **визуальная секция** под блоком цели, с лёгким `rounded-md border border-stone-700/80 bg-stone-900/40`, единый вертикальный ритм `space-y-4`.

## 8. План внедрения
