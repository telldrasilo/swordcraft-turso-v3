# План: UI прогресса строительства алтаря vs крафт / верстак / ремонт

**Статус:** план (на согласование).  
**Цель:** после старта фазы у игрока фокус на одной понятной полосе прогресса, визуально и по паттернам согласованной с остальной кузницей.

## 2. Эталоны в проекте (что изучить)

### 2.1 Крафт v2 — сегментированная полоса

Файл: [`src/components/forge/craft-v2/craft-progress.tsx`](../../src/components/forge/craft-v2/craft-progress.tsx)

- Контейнер: `rounded-lg overflow-hidden border border-stone-700`, сегменты в **одну линию flex**, высота сегмента **`h-8`**.
- **Пропорции ширины:** по доле `calculatedDuration / totalDuration` (как у алтаря по времени — тот же принцип).
- **Состояния:** завершён — сплошной цвет категории (`CATEGORY_COLORS`); в процессе — базовая заливка + **внутренний** `motion.div` на `stage.progress%`; ожидание — `bg-stone-700`.
- **Активный сегмент:** `ring-2 ring-amber-400 ring-offset-1 ring-offset-stone-900`.
- **Центр сегмента:** иконка ✓ / `Loader2` спиннер / `Circle`.
- **Tooltip:** название этапа, категория, секунды, при активном — `% выполнено`.
- Рядом: легенда категорий (бейджи), карточка текущего этапа, общий `Progress`, лог этапов.

**Вывод для алтаря:** имеет смысл перенять **высоту полосы**, **кольцо активного сегмента**, **иконки статуса в сегменте** и **тултип с длительностью + для текущего — оценка процента** (из `elapsed/durationSec`).

### 2.2 Очередь верстака (ремонт / перековка)

Файл: [`src/components/forge/workbench-queue-segmented-bar.tsx`](../../src/components/forge/workbench-queue-segmented-bar.tsx)

- Высота контейнера **`h-10`**, фон `bg-stone-950`, рамка `border-stone-700/80`.
- Сегменты — **кликабельные** кнопки, ширина в **%** от очереди; активная задача — **золотой вертикальный градиент**; остальное — `hsl` по id оружия.
- **Popover** с превью (`WeaponInventoryCard`) — для алтаря не нужен тот же контент, но паттерн «сегмент = интерактивная зона с подсказкой» совпадает с идеей тултипов крафта.

**Вывод для алтаря:** опционально чуть увеличить **hit-area** сегментов для тач/мыши; превью не требуется.

### 2.3 Панель этапов ремонта

Файл: [`src/components/ui/repair-stage-progress-panel.tsx`](../../src/components/ui/repair-stage-progress-panel.tsx)

- Карточка: `border-amber-800/50 bg-stone-950/60` — **ближе к текущему «активному» блоку алтаря** (`amber-800/35 bg-amber-950/15`).
- Текущий этап: отдельный **`Progress` (shadcn)** на весь прогресс **внутри** этапа (`h-2`).
- Список этапов с теми же тремя состояниями (галочка / спиннер / пустой круг).

**Вывод для алтаря:** вторичная полоса «только текущий микроэтап» (как в ремонте) может **дублировать** визуально сегментную полосу; либо оставить один источник правды (сегменты) + текст ETA, чтобы не раздувать UI.

## 3. Целевое поведение (продукт)

1. **До старта:** материалы, техники горна, кнопка «Начать» — как сейчас.
2. **После старта:** материалы и техники **не показывать**; в центре — **полоса прогресса**, согласованная с крафтом по читаемости и «языку» (высота, фокус, иконки, тултипы).
3. **Отмена** и идентификация фазы (заголовок / «Строительство: фаза N») остаются доступными.

## 4. План работ (этапы реализации)

### Этап A — Визуальная унификация сегментной полосы (без общей библиотеки)

- Поднять высоту сегментов алтаря до **`h-8`** (или **`h-9`** как компромисс с верстаком `h-10`), убрать «стеклянный» `gap-0.5` между сегментами или заменить на **`border-r border-stone-900`** как в крафте.
- Активный сегмент: **`ring-2 ring-amber-400 ring-offset-1 ring-offset-stone-900`** (как `ProgressSegment` при `isInProgress`).
- Внутри активного сегмента: заливка слева направо по **`elapsed / durationSec`** тем же цветом/градиентом, что и «завершённые» сегменты (либо одна палитра этапов стройки вместо градиента amber–violet для всей полосы — решение ГД).
- Центрировать в сегменте иконки: **CheckCircle2** / **Loader2** / **Circle** по образцу крафта.
- **Tooltip** на каждый сегмент: название микроэтапа, `durationSec` с, для текущего — **`~N%`** или секунды остатка.

### Этап B — Чипы этапов под полосой

- Вариант **1 (минимализм):** убрать дублирующий ряд чипов, оставить только полосу + тултипы + одну строку «Сейчас: …».
- Вариант **2 (как сейчас):** оставить чипы, но визуально связать с сегментами (нумерация, те же цвета состояния).

Рекомендация плана: после этапа A оценить на плейтесте; при избытке шума — сократить до варианта 1.

### Этап C — Общий компонент (опционально, технический долг)

- Вынести «сегментную полосу по долям времени с тремя состояниями» в **`src/components/shared/segmented-duration-bar.tsx`** (или `forge/`) с пропсами: `segments: { id, label, durationSec }[]`, `activeIndex`, `activeElapsedSec`, `renderTooltip?`.
- Подключить из **алтаря** и по желанию **сузить** дублирование с `craft-progress` (там дополнительно категории и framer-motion — можно оставить обёртку).

### Этап D — Документация и AGENTS

- После внедрения: краткая ссылка из [`ALTAR_BUILDING_V2.md`](./ALTAR_BUILDING_V2.md) или [`DEVELOPMENT_HUB.md`](./DEVELOPMENT_HUB.md) на этот файл + указание компонента в коде.
- При появлении общего компонента — одна строка в [`AGENTS.md`](../../AGENTS.md) в разделе ключевых файлов (если заведёте раздел «UI кузницы»).

## 5. Риски и ограничения

- **AltarStage** не имеет `category` как у `CraftStageInstance` — палитра по фазам или нейтральная ''stone + amber active''.
- Тики алтаря на **wall-clock** (`activePhaseStageStartTime`) — процент в тултипе должен совпадать с логикой [`computeAltarConstructionTick`](../../src/lib/altar/altar-construction-phase.ts).
- Не ломать **a11y**: сохранить `aria-label` на полосе (как у `WorkbenchQueueSegmentedBar`: `role="img"` + описание).
