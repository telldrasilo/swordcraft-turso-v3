## Документ квеста: «Эхо забытой кузни» (расширенная версия)

**Версия:** 2.0 (интеграция со строительством алтаря)  
**Статус:** финальная концепция, готова к реализации  

---

### 1. Карточка квеста (для UI «Особые задания»)

> Архивариус нашёл чертёж древнего алтаря зачарований. Его части разбросаны по опасным землям. Вам предстоит не только собрать артефакты, но и построить алтарь в кузнице, освоив новые техники. Квест проведёт вас через экспедиции, крафт и строительство.

---

### 2. Синопсис (для разработчиков)

Квест состоит из 16 шагов (0–15).  
- Шаги 0–6: сбор трёх артефактов (как в оригинале) + выдача материальных предметов `resonator_matrix`, `focusing_chalice`, `lunar_tuning_fork`.  
- Шаг 7: получение чертежа и разблокировка фазы I строительства.  
- Шаги 8–9: завершение фаз I и II, затем активная пауза с требованием выполнить крафт оружия для «расшифровки свитков».  
- Шаги 10–12: последовательное получение трёх техник через экспедиции (руны, обжиг, настройка частот).  
- Шаг 13: завершение фазы III и получение `spirit_blessing` через экспедицию.  
- Шаги 14–15: завершение фаз IV и V, финал квеста, открытие модуля зачарований.  

Все диалоги происходят в чате с выбором ответов игрока (варианты не блокируют прогресс, влияют только на тон).  
На вкладке «Алтарь» во время квеста отображается строка «Текущая цель квеста».

---

### 3. Технические обозначения

- **[Экспедиция: location]** – игрок отправляет искателей в указанную локацию через интерфейс гильдии. После успеха квест автоматически переходит к следующему шагу (если не указано иное).  
- **[Крафт оружия]** – игрок должен успешно завершить любой крафт v2 (один готовый меч). Система отслеживает событие `craftCompleted`.  
- **[Строительство: фаза N]** – игрок должен завершить все микроэтапы указанной фазы на вкладке «Алтарь». После завершения фазы генерируется событие `altarPhaseCompleted(phaseNumber)`.  
- **[Выдать предмет: id]** – добавить `materialId` в `materialStash` (с флагом `isQuestItem: true`, не расходуется).  
- **[Выдать технику: id]** – добавить `techniqueId` в `unlockedTechniques`.  
- **[Флаг шага]** – `forgottenForgeQuestStep` (0..15).

### 6. Технические флаги и интеграция

**Zustand store (добавить):**

```typescript
interface QuestForgottenForgeState {
  step: number;                       // 0..18, где 18 = завершён
  waitingForCraftAfterPhase2: boolean;
}

// Дополнительные флаги, уже существующие:
// altarUnlockedByForgottenForgeQuest (устанавливается в шаге 7)
// altarBuiltInForge (устанавливается в шаге 17)
```

**События, на которые подписывается квест:**

- `expeditionCompleted(locationId, questTag?)` – после успеха экспедиции проверяется текущий шаг и locationId, чтобы перейти к следующему шагу (шаги 1-6, 10-14). Для уникальности квестовые экспедиции могут иметь специальный тег или отдельный `questExpeditionId`.
- `craftCompleted()` – используется на шаге 9.
- `altarPhaseCompleted(phaseNumber)` – используется на шагах 8,9,13,14,15.

**UI вкладки «Алтарь»:**  
- Пока `step < 18` и `altarBuiltInForge === false`, отображать панель «Текущая цель квеста» с текстом из таблицы выше (или более короткой версией).  
- После завершения квеста панель убирается.

### 7. Примечания по реализации для разработчиков

- **Экспедиции для шагов 10,11,12,14** – не требуют новых локаций, используют существующие `oak_grove_outskirts`, `red_stone_mines`, `silver_grove`, `misty_lowlands`. Для каждой создайте отдельную запись в данных экспедиций с уникальным `id` и привязкой к квесту.  
- **Ресурс `shadow_leather`** – уже существует в каталоге материалов (tier 3, дроп в `rotten_swamp`). Игрок может добыть его заранее или сходить специально.  
- **Ресурс `mist_herbs`** – существует, дроп в `misty_lowlands`.  
- **Выдача техник** – добавьте их в список `unlockedTechniques`. Техники должны быть определены в каталоге техник.  
- **Проверка наличия ресурсов для экспедиций** – используйте стандартный механизм требований к экспедиции (можно реализовать как `requiredItems`).  
- **Сохранение прогресса** – все шаги и флаги должны быть в persisted store.
