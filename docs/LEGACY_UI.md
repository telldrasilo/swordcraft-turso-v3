# Верхняя полоса макета (история ResourceBar)

## Текущее состояние (после §3.5 / фаза 1b)

В [`game-layout.tsx`](../src/components/layout/game-layout.tsx) **нет** панели с деревом / камнем / железом / углем и скоростями прироста. Вместо неё — компактная **`TopStatusBar`**: уровень, имя, золото, эссенция души, статус сохранения, прогресс опыта. Атрибут `data-tutorial="resources-bar"` оставлен на этом блоке для туториала (шаг «Ресурсы» обновлён под новую формулировку).

## Почему старый ResourceBar был legacy

- Панель отражала укрупнённые **`ResourceKey`**, а **не** полноту **`materialStash`** и каталожных id.
- Для материалов, руд и цепочек переработки она **не** была источником правды.
- **Канон** — энциклопедия и экраны от каталога / склада по id.

## Ориентиры для разработки

- Баланс и ТЗ процессинга — от **энциклопедии и таблиц id**, не от «полоски».
- Новую логику **не** вешать на агрегаты старого вида.
