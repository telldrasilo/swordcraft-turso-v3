# Техническое задание: проработка рецепта меча и Крафтовой линии

**Версия:** 1.0  
**Статус:** задание на проработку контента и данных (реализация оркестратора и UI выходит за рамки одного рецепта, но перечислена как целевое поведение).  
**Объект:** рецепт типа «меч» (пилот — базовый меч с четырьмя частями: лезвие, гарда, рукоять, навершие).

Документ самодостателен: внешние ссылки не используются.

---

## 1. Контекст и цель

### 1.1 Проблема

В игре уже есть рецепт меча с длинной последовательностью этапов обработки, формовки, сборки и отделки. Отдельно существуют техники: обработка материала (плавка руды, дерево, кожа и т.д.) и боевые приёмы ковки с **микрозадачами** — короткими именованными шагами с обязательным стабильным идентификатором и весом длительности.

Сейчас **Крафтовая линия** в интерфейсе может строиться только из выбранных техник обработки и боевых приёмов; **этапы самого рецепта** в эту полосу не попадают. В результате игрок не видит цельную картину: сценарий «как куём меч» разъезжается между таймлайном симуляции и полосой прогресса.

### 1.2 Цель ТЗ

1. Зафиксировать **канонический сценарий работ** для рецепта меча как **цепочку макроэтапов** (хребет), согласованную с уже заданной последовательностью стадий в данных рецепта.
2. Описать, как к этому хребту **насаживаются микрозадачи техник** (обработка по выбору материала, выбранные приёмы ковки), так чтобы **каждый видимый шаг на полосе** был **известным микроэтапом** с предсказуемым id и долей времени.
3. Задать **минимальный объём данных**, который нужно добавить или уточнить в рецепте меча (и при необходимости в типах этапов), чтобы дизайнеры и разработчики могли выполнить работу без противоречий.

## 2. Термины

| Термин | Значение |
|--------|----------|
| **Макроэтап** | Один узел в последовательности `stages` рецепта: тип этапа, опционально привязка к части (`material` / `target`). Соответствует шагу в симуляторе крафта. |
| **Микроэтап / микрозадача** | Мельчайшая единица на Крафтовой линии: объект с полями **стабильный `id`**, **`label` для игрока**, опционально **`hint`**, **`durationWeight`** (вес при распределении долей времени внутри группы). |
| **Хребет рецепта** | Упорядоченная цепочка макроэтапов данного рецепта; задаёт «скелет» процесса для всех проходов по этому рецепту. |
| **Насадка техники** | Микрозадачи из выбранной техники обработки материала или боевого приёма, вставляемые в согласованные позиции относительно хребта. |
| **Крафтовая линия** | Одна полоса прогресса: визуально **блоки** (фаза работы или крупный отрезок), внутри блока **подсегменты** по микроэтапам; сумма нормализованных долей времени по всем подсегментам равна единице. |
| **Фаза линии** | Одно из трёх грубых делений процесса: подготовка материала, формовка по рецепту, завершающая ковка и отделка (точные имена полей в коде: material_preparation, recipe_forming, craft_finishing). |

## 3. Состав рецепта меча (базовый)

### 3.1 Части и роли

- **Лезвие** — металл или сплав; главное свойство для урона — твёрдость.  
- **Гарда** — металл или сплав; упор на прочность.  
- **Рукоять** — дерево; упор на упругость.  
- **Навершие** — металл или сплав; вес и баланс.

Боевая часть для расчёта — лезвие.

### 3.2 Хребет: текущая последовательность макроэтапов (эталон для проработки)

Ниже перечислены стадии в порядке исполнения, как в пилотном рецепте базового меча. Это **не просьба переписать порядок без причины**, а **опора для описания микроэтапов и насадок**. Если геймдизайн меняет порядок, ТЗ обновляется.

1. Подготовка: разогрев горна, подготовка инструментов.  
2. Обработка заготовки **лезвия**: плавка (металл), прокат.  
3. Обработка заготовки **гарды**: плавка, прокат.  
4. Обработка **рукояти**: распил дерева.  
5. Обработка **навершия**: плавка.  
6. Формовка **лезвия**: нагрев, ковка формы.  
7. Формовка **гарды**: нагрев, придание формы.  
8. Формовка **рукояти**: резьба/обработка формы.  
9. Формовка **навершия**: нагрев, придание формы.  
10. Сборка: подгонка, соединение, обмотка рукояти, балансировка.  
11. Отделка **лезвия**: закалка, отпуск, шлифовка, заточка.  
12. Общая отделка: полировка, осмотр.

Для каждой строки хребта в проработке должно быть задокументировано: **к какой фазе линии относится**, **какие микроэтапы базового сценария** (см. раздел 4) и **куда встают насадки** (раздел 5).

## 4. Микроэтапы хребта рецепта

### 4.1 Принцип

Каждый макроэтап рецепта для целей Крафтовой линии должен раскладываться хотя бы на **один** микроэтап с явным **`id`**. Рекомендуется **два–четыре** микроэтапа на типичный макроэтап, чтобы полоса читалась как «ход работы», а не как один непрозрачный блок.

Идентификаторы микроэтапов хребта должны быть **стабильными** (не меняться при правках текста `label`), **уникальными внутри рецепта** и по возможности **префиксованными** именем рецепта или типа оружия, например: `basic_sword_macro_prep_heating_01`.

### 4.2 Содержательные требования по группам хребта

**Подготовка горна и места**  
Микроэтапы: разжигание или поддержание огня, контроль температуры, выкладка инструмента. Не дублировать дословно длинные описания из энциклопедии; короткие глаголы в `label`.

**Обработка металла частей (лезвие, гарда, навершие)**  
Базовые микроэтапы описывают работу **по уже имеющейся заготовке** в сценарии рецепта (плавка и прокат в списке стадий). Если игрок выбрал **путь от руды**, дополнительная **насадка техники плавки** (раздел 5.1) логически предшествует или перекрывает по смыслу первые шаги металлообработки данной части — это нужно явно решить в таблице соответствия для пилота (без двойного счёта времени и без дублирования одного и того же действия на полосе).

**Дерево рукояти**  
Базовые микроэтапы: распил, черновая обработка. При выборе заготовки «от брёвна» насадка техники распила досок согласуется с этапами `proc_sawing`.

**Формовка по частям**  
Для лезвия и металлических частей: нагрев, удары/правка, контроль геометрии. Для рукояти: перенос разметки, снятие стружки, посадка формы.

**Сборка**  
Подгонка посадок, клёпка или иная фиксация (в терминах игры — без обязательной детализации исторического метода), обмотка, проверка баланса на весах или в руке.

**Отделка**  
Закалка и отпуск лезвия — отдельные микроэтапы или сгруппированы по таблице; шлифовка, заточка; общая полировка и финальный осмотр.

### 4.3 Длительность

Для каждого микроэтапа хребта задаётся **`durationWeight`**: положительное число. Абсолютные секунды на полосе выводятся из общей модели времени крафта; на этапе проработки рецепта достаточно **согласованных относительных весов** внутри макроэтапа и по ощущению **между макроэтапами** (чтобы закалка не выглядела дешевле разогрева горна без замысла).

## 5. Насадки техник

### 5.1 Обработка материала (плавка, дерево, камень, кожа)

Если для части выбран режим «от сырья» (например руда в горне), игрок выбирает **технику обработки материала**. Её **микрозадачи** целиком попадают на Крафтовую линию в блоке фазы **подготовка материала**, **до** соответствующего участка хребта рецепта, который опирается на уже готовую заготовку этой части.

Правила пилота:

- Одна и та же техника плавки не должна отображаться дважды для одной части.  
- Порядок: сначала все блоки обработки (в порядке, согласованном с зависимостями частей или с явным порядком в данных), затем хребет рецепта по макроэтапам.  
- Цвет блока на полосе различает фазу; подсегменты внутри — микрозадачи из данных техники.

### 5.2 Боевые приёмы ковки

Игрок выбирает один или несколько **боевых приёмов** (складывание, закалка, полировка и т.д.). У каждого приёма есть свои **микрозадачи** и, как правило, **модификации процесса** (добавление или замена стадий в таймлайне).

Требования к проработке для меча:

- Для каждого **поддерживаемого** приёма в связке с этим рецептом должна быть таблица: **к какому макроэтапу или типу стадии привязаны микрозадачи приёма** (якорь).  
- Если явного якоря в данных ещё нет, для пилота допускается **временная эвристика** (например все микрозадачи приёма после первого этапа формовки лезвия), но результат нужно зафиксировать в списке «технический долг» и заменить явными полями в данных техники или рецепта.  
- Микрозадачи приёма **не** должны дублировать по тексту базовые микроэтапы хребта на том же действии, если оба источника показываются подряд без слияния — либо слияние в один подсегмент, либо разведение по времени (решение фиксируется в таблице сценариев).

### 5.3 Конфликты и приоритет

При одновременном наличии микроэтапа хребта и микрозадачи техники на одном и том же игровом событии приоритет за **явным правилом в данных**. В отсутствие правила действует порядок: **насадка техники не удаляет микроэтапы хребта**, а **дополняет или заменяет** согласно описанию приёма; противоречия выявляются на ревью сценария «базовый меч + один приём».

## 6. Крафтовая линия: визуальная и логическая модель

### 6.1 Блоки

Блок объединяет микроэтапы, относящиеся к одной **фазе линии** или к одному **выделенному макроэтапу рецепта** (выбор продукта: крупнее блок = проще читать, мельче = детальнее). Для ТЗ пилота достаточно правила: **цвет блока определяется фазой линии**; внутри фазы `recipe_forming` возможны **несколько блоков**, по одному на **логически значимую группу макроэтапов** (например «обработка всех металлических заготовок», «формовка лезвия»).

### 6.2 Подсегменты

Каждый подсегмент — один микроэтап с **уникальной парой** (контекст рецепта, `id`) в рамках сохранения прогресса. Для отображения достаточно `label` и доли ширины.

### 6.3 Согласование с симуляцией

Игрок ожидает, что движение полосы соответствует **реальному таймлайну крафта**. Требование: итоговый список микроэтапов на полосе должен быть **согласован** с порядком и длительностями исполняемых стадий (либо общий оркестратор строит оба представления, либо снапшот плана однозначно порождает линию и проверяется тестом). Дополнимое отставание UI от симуляции допускается только как известный дефект с планом устранения.

## 7. Объём работ по данным (чеклист)

### 7.1 Матрица «часть × источник материала» (`basic_sword`)

Источник исполнения таймлайна и полосы — развёрнутый план (`generateCraftStages` / `collectExpandedStageConfigsForCraft`). Насадка обработки вставляется **перед** первым макроэтапом хребта по данной части, без дубля «та же плавка» в UI.

| Часть | Режим в плане (`partMaterialSupply`) | Допустимые техники обработки (пример) | Примечание |
|--------|--------------------------------------|---------------------------------------|------------|
| `blade` | `direct` + материал-слиток/сплав в `materials` | — (блок обработки не вставляется) | Хребет начинается с `proc_smelting`/`proc_rolling` рецепта. |
| `blade` | `ore_smelt` | плавка руды под выбранный металл (например `forge_basic_iron_smelt`) | Перед слотом плавки лезвия на полосе — микрозадачи техники плавки. |
| `guard` | `direct` | — | Аналогично лезвию по металлу. |
| `guard` | `ore_smelt` | как для металла гарды | Вставка перед `proc_smelting`/`proc_rolling` для `guard`. |
| `pommel` | `direct` / `ore_smelt` | как для металла навершия | Камень/самоцвет — без плавки по железному сценарию; отдельные профили материалов — в каталоге материалов. |
| `grip` | `direct` + доски | — | Хребет — `proc_sawing` для `grip`. |
| `grip` | путь от брёвна (если включён в игре для древесины) | распил/раскрой под профиль материала | Насадка согласуется с `proc_sawing`; порядок — в общем таймлайне до соответствующего слота. |

### 7.2 Матрица «рецепт × боевой приём» (`basic_sword`)

| Приём (`id`) | Якорь на линии | Модификация процесса (`processMods`) | Заметка |
|----------------|----------------|----------------------------------------|---------|
| `basic_forging` | `craftLineAnchorAfterStageIndex: 8` (после `form_forging` лезвия в базовом хребте) | — | Микрозадачи приёма вставляются после указанной стадии **базового** рецепта. |
| `balanced_design` | *(по умолчанию — после хребта, если индекс не задан)* | `addStage`: после `asmb_joining` добавляется `asmb_balancing` | Полоса следует развёрнутому таймлайну; дубликат/доп. балансировка отображается синтетическим сегментом, если слота нет в исходных микрошагах. |
| `double_hardening` | *(хвост или явный якорь при появлении в данных)* | `addStage`: после `fin_hardening` ещё один `fin_hardening` | Для UX желательно позже зафиксировать `craftLineAnchorAfterStageIndex` у техники. |

Неподдерживаемые пары не перечисляются здесь: при появлении приёма с `replaceStage` / конфликтующими вставками — отдельная строка в этой таблице и тест в `craft-line-sim-alignment`.

1. **Таблица микроэтапов хребта** для рецепта базового меча: для каждого макроэтапа из раздела 3.2 — список `id`, `label`, `durationWeight`, принадлежность к фазе линии (частично закрыта данными `basic-sword-stages.ts`).  
2. **Матрица «часть × источник материала»**: см. **§7.1**; уточнять при новых материалах/профилях обработки.  
3. **Матрица «рецепт меча × боевой приём»**: якорение и `processMods` — см. **§7.2**; перечень неподдерживаемых пар с причиной пополнять по мере контента.  
4. **Единый реестр идентификаторов** микроэтапов хребта этого рецепта (чтобы не пересечься с id микрозадач техник — разные префиксы).  
5. **Копирайт**: вычитка `label` на единый тон (коротко, без дублирования названия рецепта в каждой строке).  
6. **Приёмочный сценарий**: один полный проход «базовый меч, руда на лезвии, одна техника плавки, два боевых приёма» — текстовое описание ожидаемой последовательности подписей на полосе сверху вниз.

## 8. Нефункциональные требования

- Идентификаторы пригодны для **будущего персиста** (открытые микроэтапы, тома изучения): не привязывать смысл только к порядковому номеру в массиве без id.  
- Любое изменение хребта после релиза контента сопровождается **миграционной заметкой** для сохранений, если по id копятся достижения игрока.  
- Документ ТЗ **обновляется**, если меняется порядок стадий в рецепте или состав частей пилотного меча.

## 9. Критерии приёмки проработки

- По рецепту базового меча существует полное описание микроэтапов хребта с уникальными `id` и весами.  
- Описаны все обязательные сочетания выбора сырья для четырёх частей с точки зрения порядка блоков обработки и хребта.  
- Для не менее **трёх** боевых приёмов, реалистичных для базового меча, зафиксировано якорение микрозадач относительно макроэтапов или стадий.  
- Нет неразрешённых дублей одного действия на полосе без явного правила слияния.  
- Зафиксирован минимум один сквозной сценарий с ожидаемой последовательностью подписей на Крафтовой линии.

## 10. За рамками данного ТЗ (но учитывать при оценке)

- Реализация в коде обобщённого сборщика Крафтовой линии из хребта рецепта плюс техники.  
- Расширение типов `WeaponRecipe` или реестра стадий полями для микроэтапов и якорей.  
- Полировка общего UI компонента полосы (отступы, легенда, доступность).  
- Масштабирование на **много рецептов** без копирования длинных `stages`: модуль **шаблонов и фрагментов** с материализацией в тот же `WeaponRecipe` — проектная спецификация **`docs/RECIPE_TEMPLATE_COMPOSITION.md`** (после пилота эталон `basic_sword` может быть воспроизведён шаблоном для CI-сравнения).

На этом ТЗ по проработке рецепта меча и согласованной с ним Крафтовой линии считается исчерпанным.
