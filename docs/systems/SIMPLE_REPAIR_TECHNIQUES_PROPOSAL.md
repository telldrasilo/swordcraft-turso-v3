# Простой широкий ремонт и отказ от авто-починки за золото — согласованный дизайн (черновик под реализацию)

Документ фиксирует аудит данных, **принятые решения** по ранней игре и лейтовым стихиям, целевой **риск шрамов ~30%** для грубого применения базовых техник (уточняется в коде), а также **что делают** опции «Дополнительно: осмотр углём и авто-ремонт».

Источники правды в коде: `src/data/weapon-damage/damage-tag-registry.ts`, `repair-techniques-registry.ts`, `src/store/cross-slice/repair-cross-slice.ts`, `src/components/ui/repair-card.tsx`.

---

## Принятые решения (ответы на вопросы согласования)

| # | Решение |
|---|---------|
| 1 | **Не плодить дубли:** если новая группировка совпадает с уже существующей базовой техникой — **дописать теги в неё** (`clearsTagIds`). У кузнеца **с самого старта** должны быть **самые простые базовые** техники починки (tier `basic`, `unlock: 'base'`), без обязательной гильдии. |
| 2 | Ровно **5** игровых «опорных» техник в базовом слое (см. §3.2): четыре снимают теги + одна про прочность. |
| 3 | **Коррозия и влага** (`elemental_corrosion_rot`, `elemental_water_soak`) — **в общем раннем стихийном кластере**, не отдельная техника. |
| 4 | **Лейтовые стихии** — только **узкоспециализированные** техники (покупка у интенданта / прогресс): **`elemental_skverna_taint`**, **`elemental_space_shift`**, **`elemental_arcane_noise`**, **`elemental_darkness_stain`** (бездна). В базовый «простой» слой **не входят**. Ранняя игра опирается на базовые техники; без гильдии игрок всё равно может жить за счёт п.1. |
| 5 | **Fallback «только прочность»** (`durability_maintenance`) — **да**, оставить; эффект **умеренный** («немного»): не заменяет полноценный ремонт по тегам, дешёвый сервисный проход без снятия повреждений. |

## 2. Текущие техники ремонта (краткий аудит)

Файл: `repair-techniques-registry.ts`.

**Базовые (`repairTier: basic`)** сейчас:

- `durability_maintenance` — только прочность, без тегов.
- `edge_truing` — `slash_chip`, `blunt_dull`, `puncture_gouge`.
- `haft_tightening` — `loose_fitting`, `handle_split`.

**Узкоспециализированные** — для точной работы и лейта; после согласования часть тегов **переносится в базовые** (см. §3), узкие остаются как **низкий риск шрамов** / узкий охват / лейт.

**Авто-ремонт за золото** — по-прежнему планируется убрать как основной обход (см. §5).

## 3. Пять базовых техник (целевая схема)

Принцип: **нет отдельных «широких клонов»** рядом с `edge_truing` / `haft_tightening` — только **расширение `clearsTagIds`** и **до двух новых id** с `repairTier: basic`, чтобы закрыть раннюю игру и заменить золотую автопочинку.

Целевой шанс шрамов при «грубом» базовом проходе для согласования: **~30%** (реализация: множитель к `incrementScarWeightsFromClearedTags` или отдельный бросок — отдельная задача).

### 3.1 Техника 1 — расширить **`edge_truing`** (правка кромки и удара по полотну)

**Добавить теги:** `physical_gouge_chunk`, `physical_impact_dent` (к уже существующим трём).

Итого снимает: **5** физических тегов — сколы, затупление, прокол, зарубина глубокая, вмятина.

*Узкие* `notch_filing` и т.п. остаются для игроков, которые хотят **меньше шрамов** на тех же тегах.

### 3.2 Техника 2 — расширить **`haft_tightening`** (узел и посадка)

**Добавить тег:** `physical_bend_warp` (перегиб — в базовой «полевой» перетяжке вместе с рукоятью/крепом).

Итого снимает: **3** тега — люфт, трещина рукояти, перегиб.

*Узкая* `guard_straightening` / `axis_straightening` — прецизионный вариант с меньшим риском.

### 3.3 Техника 3 — **новая базовая** (рабочее имя на согласование, например «Снятие усталости металла»)

**Новый `id`** (`repairTier: basic`, `unlock: 'base'`): снимает **`physical_crack_fissure`** и **`elemental_frost_bite`** одним проходом (трещина + мороз как связанный стресс металла).

`elemental_frost_bite` **не дублировать** во втором стихийном кластере (§3.4), чтобы один тег не снимался двумя базовыми техниками с разной экономикой.

### 3.4 Техника 4 — **новая базовая** «ранний стихийный уход» (рабочее имя)

**Новый `id`** (`repairTier: basic`, `unlock: 'base'`): снимает **все «ранние» стихии** — **10 тегов**:

`elemental_fire_scorch`, `elemental_water_soak`, `elemental_earth_grit`, `elemental_air_shear`, `elemental_lightning_arc`, `elemental_poison_etch`, `elemental_light_sear`, `elemental_nature_bloom`, `elemental_blood_mark`, `elemental_corrosion_rot`.

**Не входят:** четыре лейтовых + скверна (§1.2); **не входит** `elemental_frost_bite` (снятие в §3.3).

*Узкие* `elemental_prima_treatment`, `elemental_veil_treatment`, `corrosion_cleanup` и др. остаются как **точечные / низко-шрамные** альтернативы.

### 3.5 Техника 5 — **`durability_maintenance`** (только прочность)

Без снятия тегов. Баланс: **небольшой** прирост прочности / «подтянуть до похода», без замены ремонта по тегам. Стоимость — по текущей логике экономики ремонта; при реализации не раздувать до уровня полноценного восстановления.

### 3.6 Сводка покрытия

| Базовая техника | Тегов снимает | Примечание |
|-----------------|---------------|------------|
| `edge_truing` (расш.) | 5 физ. | |
| `haft_tightening` (расш.) | 3 физ. | |
| новая (трещина + мороз) | 1 физ. + 1 стих. | без дубля мороза в п.4 |
| новая (ранние стихии) | 10 стих. | вода и коррозия внутри |
| `durability_maintenance` | 0 | только прочность, «немного» |

**Лейт только узкими:** `elemental_space_shift`, `elemental_darkness_stain`, `elemental_arcane_noise`, `elemental_skverna_taint` — см. существующие specialized / будущие привязки в `allowedRepairTechniqueIds` у тегов и интендант.

## 4. Интендант и гильдия

- Базовые **пять** не требуют покупки за репутацию (стартовый набор кузнеца).
- Узкие техники — за репутацию, для снижения шрамов и закрытия лейта.
- При необходимости дублировать предложения в `intendant-catalog.ts` с пометкой «уже есть с нуля» только для подсказки UI — по усмотрению реализации.

## 5. Замена авто-ремонта за золото

Текущая логика: `claimWeaponAutoRepair` — золото, частичная прочность, снятие всех тегов, штрафы к `epicMultiplier`, шрамы.

**План:** убрать как основной способ; игрок опирается на §3. Детали миграции сейвов / `STORE_VERSION` — при внедрении.

## 6. Блок «Дополнительно: осмотр углём и авто-ремонт» (факт по коду)

### 6.1 «Осмотр углём»

Глубокий осмотр **за уголь** (кнопка «Осмотреть глубже (уголь ×N, T с)»): `startWeaponDeepInspect` / `completeWeaponDeepInspect`, заметки `deepInspectNotes`. Это **подсказки и мета осмотра**, не отдельный «кубик» ремонта.

### 6.2 Авто-подбор техник

«Кузнец подберёт техники» — пропуск квиза, наценка на материалы, режим диагностики `auto_pick` (хуже для будущей меты зачарования, чем ручной осмотр).

### 6.3 Авто-ремонт за золото

`claimWeaponAutoRepair` / очередь на кузницу — быстрый обход с штрафами; подлежит удалению как замена §3.
