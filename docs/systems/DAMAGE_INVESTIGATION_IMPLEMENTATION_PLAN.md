## Фаза 4 — UX: модалка, инвентарь, вкладка «Ремонт»

**Статус:** порядок вкладок кузницы **Крафт → Инвентарь → Ремонт → Магазин** зафиксирован в коде. **Инвентарь:** краткий счётчик «нужен уход» + переход во вкладку «Ремонт» (`navigateToForgeTab` / `forgeTabRequest`); на карточке оружия — **«Отправить на ремонт»** (ставит оружие в слот верстака на вкладке «Ремонт»), плюс переход по прочности/тегам. **Вкладка «Ремонт»:** один слот верстака (`repairBenchWeaponId` в store): выбранное оружие показывается в `RepairCard` с `variant="repairWorkbench"` — без блока «Накопленные свойства», краткие подписи тегов; экономика — **материалы и время этапов**, **золото за ремонт не списывается**. **Глубокий осмотр:** материалы + таймер (`deepInspectReadyAt` / `weaponLegacy`). **Пост-миссия:** в модалке награды при видимых повреждениях — ссылка «Открыть вкладку „Ремонт“ в кузнице»; тексты G2/G3 без отсылки к «классическому ремонту».

**Цель:** игрок видит повреждения после миссии и в кузнице **без дублирования**: инвентарь — кратко; **ремонт** — подробно и по делу (концепт §12).

**Работы:**

- Модалка награды / блок после миссии: текст повреждений, клик к журналу.
- Журнал миссии: строка о фиксации повреждений.
- **Навигация кузницы:** порядок вкладок — **… → Инвентарь → Ремонт** (`forge-screen` / layout).
- **Инвентарь:** иконка + 1–2 тега максимум; без полной копии карточки ремонта; **«Отправить на ремонт»** заполняет слот верстака.
- **Вкладка Ремонт:** одно оружие в верстаке, теги, техники, план этапов (3b), авто-ремонт, осмотр — **только релевантное полю ремонта** (не «список всех повреждённых клинков» как единственный режим).
- После успешного ремонта слот верстака сбрасывается; нет «висящих» меток в инвентаре.

**Критерий готовности:** ручной сценарий «миссия → повреждения → вкладка Ремонт → починка → инвентарь чист» без противоречий; нет ощущения двух одинаковых карточек оружия.

**Зависимости:** фаза 3b для полного цикла техник; частично — фаза 3 + старый UI до замены.
